# 设计游戏局内界面 (In-Game UI Design)

**技能定位**：
专注于生成用于演示核心玩法的常规关卡图或局内界面图集。

## 1. 定义场景框架
- **顶部区域 (Top)**：进度条（消除目标、连击X3等）、关卡数字显示器、以及暂停/设置图标。
- **核心交互区 (Middle)**：占画面最大比例。呈现一组大型立体货架（如三层结构）。货架内随机摆放丰富多样的 3D 超市物品碎片（如：两个一样的汽水瓶+一个错放的蛋糕）。可安排某些商品被阴影遮挡或处于半透明的“冰冻/锁定”状态。
- **底部技能栏 (Bottom)**：通常是醒目的渐变按钮板，包含三大常用道具：例如“放大镜（提示）”、“后退/撤回箭头”、“魔术棒/刷新（打乱商品）”。

## 2. 交互重点传达
画面中应包含 UI 手势指示（如白色的卡通小手或虚线箭头），展示玩家正在将一款商品拖拽至另一个同类商品旁凑成 3 个（Match-3）。

## 3. 输出标准
根据上述框架，向用户提供高精度的 Prompt 或者详尽的画面拆解说明，甚至结合引擎生成资产图进行展示。
